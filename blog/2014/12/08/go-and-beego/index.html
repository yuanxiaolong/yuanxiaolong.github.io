GOPM 第三方的 GO包管理工具

值得说的是，go 并没有自带的 包管理工具，这对于 java ,ruby ,nodejs这样的同学来说，就感到十分困惑了。

这里推荐一个包管理工具 gopm 地址，你可以在开发环境使用它，而到生产环境时 就用 go get来获取依赖。

使用gopm有3个好处

  1. 它隔离了GOPATH在当前的工程，以 .vendor 文件夹的形式存在
  2. 有一个内容描述文件 .gopmfile，但这个文件不像maven的 pom.xml ，为什么？因为 go 获取依赖包，最终是根据代码里的 import来的,所以此文件仅展示用.
  3. gopm list可以快速展现当前工程依赖了哪些包

当然缺点就是，它没有像 beego 的 bee run这样有热加载能力
